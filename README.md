# prodv_C-

1.1) Инкапсуляция.
  При инкапсуляции объекты могут управлять своим состоянием через внутренние методы, невидымые для пользователя.
  2) Наследование.
  Позволяет использовать реализацию уже написанного класса в новом. Сделано для уменьшения количества кода при наличии общих черт у классов.
  3) Полиморфизм.
  Виртуальные функции позволяют обобщать функции, написанные для различных унаследованных классов.
  4) Абстракция.
  Позволяет пользователю не задумываться о реализации класса/функции/метода, а просто пользоваться.
